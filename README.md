# 豆包语音跟随

这是一个原生 Android 应用，用麦克风音量判断用户是否正在说话，并通过无障碍手势按住豆包聊天页底部的“按住说话”按钮。

## 工作逻辑

- 只在豆包处于前台时激活，切换到其他应用会立即松手并暂停触发。
- 检测到人声后按住“按住说话”。
- 用户可以设置“低于多少音量算说完”，范围为 0.2%–3.0%，默认 0.6%。
- 用户可以设置“低音量持续多久松开”，范围为 0.2–2.0 秒，默认 0.4 秒。
- 当实时音量低于设置值并持续达到设置时间后，立即松开发送。
- 如果豆包抢占麦克风导致音频读取完全停住，仍保留 2 秒兜底松手，避免一直按住。
- 豆包播放回复时暂停触发，避免把豆包自己的声音当成用户说话。
- 麦克风灵敏度可在应用首页设置为 1–10 级，设置会实时生效。
- 无障碍服务连接后自动显示悬浮控制，悬浮控制保留“启用麦克风跟踪”和“结束跟踪”两个按钮。

## 首次使用

1. 安装稳定签名 APK。
2. 打开“豆包语音跟随”，点击“完成首次设置”。
3. 允许麦克风权限。
4. 在系统无障碍页面找到“豆包语音跟随”，手动开启一次。
5. 返回应用，点击“打开豆包”。
6. 进入豆包聊天页并切换到“按住说话”模式，再点击悬浮窗里的“启用麦克风跟踪”。

Android 不允许普通应用自行打开无障碍权限，因此第一次必须由用户在系统页面确认。此工程使用固定签名构建，以后的 APK 可以直接覆盖安装，正常更新或手机重启不会重复清除该授权。

## 灵敏度

- 1–3：适合环境较吵、容易误触发的场景。
- 4–7：日常使用，默认是 6。
- 8–10：适合说话声音较小或手机距离较远的场景。

悬浮窗会实时显示当前音量、触发阈值和灵敏度。数值越高，触发阈值越低。

## 说完判定

- 结束音量调高：更容易快速判断说完，适合较吵的环境，但过高可能截断较轻的尾音。
- 结束音量调低：更能保留较轻的尾音，但背景声高于该值时可能不松手。
- 松手时间调短：反应更快。
- 松手时间调长：允许句子中间有更长停顿，不容易提前发送。

这两个设置和麦克风灵敏度一样，会在跟踪过程中实时生效。

## 构建

GitHub Actions 使用以下仓库密钥生成固定签名 APK：

- `ANDROID_KEYSTORE_BASE64`
- `ANDROID_KEYSTORE_PASSWORD`
- `ANDROID_KEY_ALIAS`
- `ANDROID_KEY_PASSWORD`

构建产物名称为 `DoubaoVoiceFollower-stable-apk`，内部文件为 `app-release.apk`。

## 隐私

麦克风数据只在手机本地计算音量，不保存录音，也不上传音频。
