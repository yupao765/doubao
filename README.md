# 豆包麦克风跟踪

这是一个原生 Android 工程。应用启动后会显示授权页；授权完成后可以打开一个悬浮窗，悬浮窗只有两个按钮：

- 启用麦克风跟踪
- 结束跟踪

启用后，应用会在本机监听麦克风音量变化。只有豆包处于前台时，检测到人声才会通过 Android 辅助功能服务按住豆包聊天页底部的“按住说话”按钮；连续约 2 秒没有检测到人声后松开。松开计时由独立计时器处理，即使豆包抢占麦克风导致录音读取停顿，也会按时松开。手机正在播放豆包回复时会暂停触发，避免豆包自己的声音被当成用户说话。

## 使用步骤

1. 用 Android Studio 打开本目录。
2. 连接安卓手机，构建并安装 App。
3. 首次打开 App，依次开启：
   - 麦克风权限
   - 辅助功能服务：豆包麦克风跟踪
4. 点击“显示悬浮按钮”，或重新进入豆包时使用已经显示的悬浮按钮。
5. 打开豆包，进入你截图里这种聊天页，底部显示“按住说话”。
6. 点击悬浮窗里的“启用麦克风跟踪”。
7. 需要停止时，点击悬浮窗里的“结束跟踪”。

## 没有 Android Studio 时在线打包

推荐用 GitHub Actions：

1. 在 GitHub 新建一个空仓库。
2. 把本目录里的所有文件上传到仓库根目录，包含隐藏目录 `.github`。
3. 打开仓库的 `Actions` 页面。
4. 选择 `Build APK`。
5. 点击 `Run workflow`。
6. 等构建完成后，进入本次运行页面，在 `Artifacts` 里下载 `DoubaoMicTracker-debug-apk`。
7. 解压后得到 `app-debug.apk`，把它发到手机上安装。

这是调试签名 APK，可以直接手动安装测试；如果以后要上架应用商店，再做正式签名版。

## 关键说明

- 麦克风音频只在本机做音量/人声活动判断，不保存录音。
- 悬浮按钮使用辅助功能服务的 `TYPE_ACCESSIBILITY_OVERLAY`，不是普通 App 悬浮窗。这样更符合 Android 对“辅助功能服务 + 普通 App”共享麦克风输入的规则。
- 悬浮窗会显示当前状态和实时音量，例如 `音量 3% / 阈值 1%`。只有音量达到阈值时才会按住豆包按钮。
- 当前台不是豆包时，悬浮窗会显示 `等待豆包前台`，并且不会触发任何按压。前台检测会同时看包名和豆包聊天页特征，例如 `按住说话`、`内容由 AI 生成`。
- 说话停止后的松开由独立计时器触发，不依赖麦克风持续返回静音数据。释放手势使用定时续接和最后抬手片段，悬浮窗会短暂显示 `正在松开发送`。
- 程序优先在当前界面寻找包含“按住说话”的辅助功能控件。
- 如果当前豆包版本没有暴露这个控件文本，会兜底按屏幕底部中央位置，适配你发来的界面结构。
- 豆包 Android 包名优先按 `com.larus.nova` 判断；如果厂商渠道包名不同，会用界面里的 `按住说话` + 豆包文案特征兜底识别。
- 如果手机系统提示后台麦克风被限制，请先回到本 App 前台，再点击悬浮窗“启用麦克风跟踪”。
- Android 官方说明：<https://developer.android.com/media/platform/sharing-audio-input>

## 可调参数

- 无声松开时间：`VoiceTracker.SILENCE_RELEASE_MS`，默认 2000 毫秒。
- 人声阈值：`VoiceTracker.MIN_SPEECH_RMS` 和 `VoiceTracker.AMBIENT_MULTIPLIER`。默认逻辑是实时音量同时超过最低人声音量和环境底噪倍数，环境太吵时可以调高，检测太迟钝时可以调低。
- 兜底按压位置：`DoubaoAccessibilityService.findDoubaoPressTarget()` 里的 `screen.height - dp(64)`。
